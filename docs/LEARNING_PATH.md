## Блок 0 — Орієнтація в Godot (0.5–1 день)

Мета: не боятися інтерфейсу.

1. **Що таке Node, Scene, SceneTree.**
   - Node — "деталь". Scene — "збірка деталей" (`.tscn`). SceneTree — "все разом у грі".
2. **Інспектор, Scene dock, FileSystem dock.**
3. **Як додати Node, перейменувати, зберегти сцену.**
4. **Запуск проєкту (F5) vs запуск сцени (F6).**

**Перевірка:** створи сцену з `Node2D` + `Sprite2D`, присвой спрайт `icon.svg`, запусти — бачиш іконку на екрані.

---

## Блок 1 — GDScript базовий (1 день)

Мета: читати і писати скрипти з типами.

1. Змінні, типи (`int`, `float`, `String`, `bool`, `Vector2`).
2. Функції, `_ready()`, `_process(delta)`.
3. `@export` — видимі змінні в інспекторі.
4. `@onready var node: Type = $Path`.
5. `if`/`for`/`while`, масиви (`Array`), словники (`Dictionary`).
6. **Статична типізація — обов'язкова** (див. copilot-instructions §5).

**Перевірка:** `Sprite2D` рухається по синусоїді; швидкість і амплітуду крутиш в інспекторі.

---

## Блок 2 — TileMapLayer і грід (M1) (1 день)

Мета: відобразити першу карту-сітку табору.

1. `TileSet` — що це, як нарізати спрайт-атлас на плитки.
2. `TileMapLayer` (Godot 4.6) — як додати шар, намалювати поле 20×20.
3. Перетворення координат: `local_to_map()`, `map_to_local()`.
4. Власний модуль `tile_grid.gd` — обгортка над координатами (`grid_to_world`, `world_to_grid`).
5. Підсвітка клітинки під курсором (`Sprite2D` + `_process`).

**Перевірка:** поле 20×20 зелених плиток, миша по ньому — підсвітка клітинки під курсором.

---

## Блок 3 — Камера і меню (0.5 дня)

1. `Camera2D` (у стратегії — **не** слідкує за гравцем, гравець керує нею сам).
2. Pan (drag правою кнопкою), zoom (коліщатко).
3. Обмеження камери межами карти.
4. Перехід між сценами: `main_menu.tscn` → `game.tscn` → `game_over.tscn`.

**Перевірка:** з меню запускаєш гру, рухаєш камеру мишею і клавіатурою, повертаєшся в меню через ESC.

---

## Блок 4 — Сигнали і компоненти (1 день)

Мета: зрозуміти, як ноди спілкуються без жорстких зв'язків.

1. Що таке **сигнал** (подія, яку видає нода).
2. Підключення сигналу в редакторі vs у коді (`node.signal_name.connect(callable)`).
3. Створення свого сигналу: `signal died`.
4. **Pattern: Health Component.**
   - `health_component.gd` з `max_health`, `current_health`, сигналами `health_changed`, `died`.
   - Чіпляється до героя, ворога і будівлі — **reuse**.

**Перевірка:** у героя, ворога і стіни **один і той самий** скрипт здоров'я.

---

## Блок 5 — Resource-файли для даних (M2) (1 день)

Мета: відокремити дані балансу від коду. Найголовніший патерн стратегії.

1. Що таке `Resource` і чим відрізняється від `Node`.
2. `class_name BuildingData extends Resource` з `@export`-полями: `id`, `size: Vector2i`, `hp`, `cost: Dictionary`, `sprite`, `name_loc_key`.
3. Створення `.tres` через FileSystem → New Resource → BuildingData.
4. `BaseBuilding.gd` зчитує `@export var data: BuildingData`.
5. Створити 2 будівлі (**Стіна**, **Сільрада**) — **без** зміни коду, лише 2 `.tres`.

**Перевірка:** у інспекторі міняєш HP у `wall.tres` — у грі стіна має інший HP без релоаду коду.

## Блок 6 — Розстановка будівель на гріді (M2) (1 день)

1. `Control`-панель з кнопками "Побудувати Х".
2. Режим розстановки: привʼязана до курсора "привид" будівлі.
3. Клік → перевірка "чи вільно на цих плитках" → `instantiate` у сцену.
4. Відміна (ПКМ або ESC).

**Перевірка:** відкриваєш панель, тицяєш "Стіна", клікаєш на карті — зʼявляється стіна з hp-баром.

---

## Блок 7 — Autoload, ResourceStore і EventBus (1 день)

Мета: глобальні системи, до яких може звертатися будь-хто.

1. Що таке **autoload (singleton)** — Project Settings → Autoload.
2. `ResourceStore.gd` — словник ресурсів (`wood`, `metal`, `food`, `ammo`, `money`, `energy`, `coal`) + сигнал `resource_changed`.
3. `EventBus.gd` — глобальні сигнали (`tile_revealed`, `building_placed`, `wave_started`, `hero_died`).
4. Правило: **UI слухає, логіка пише.** HUD підписаний на `resource_changed`, а будівля викликає `ResourceStore.spend(...)`.

**Перевірка:** поставив стіну — HUD сам оновив "Деревина: -10".

---

## Блок 8 — Відкриття плиток і туман війни (M3) (1 день)

1. Два `TileMapLayer`: `ground` і `fog`.
2. Масив `Array[Vector2i]` — які плитки відкриті.
3. Клік по затемненій сусідній плитці → `Timer` на "розчищення" → зняти з fog.
4. Генерація вмісту плитки (ресурс / ворог / пусто) — випадково на старті гри або при відкритті.

**Перевірка:** на старті видно 3×3 навколо Реактора. Розчистив плитку — відкрилася нова, на ній дерево/брухт.

---

## Блок 9 — Герої та pathfinding по гріду (M6) (1.5 дня)

1. `AStarGrid2D` — вбудований A* у Godot 4 для сіток.
2. Побудова графа: `region`, `cell_size`, `update()`, `set_point_solid(...)` для зайнятих плиток.
3. `hero.tscn` — `Node2D` + `Sprite2D` + `Area2D` (для кліку).
4. Клік по плитці → `get_id_path()` → герой анімовано рухається по точках.
5. Виділення героя (рамка/контур).

**Перевірка:** клікаєш на героя, потім на плитку — він іде в обхід стін.

---

## Блок 10 — Автобій: вежі, герої, projectile-и (M7, M9) (1.5 дня)

1. `Area2D` як "сенсор дальності" — знаходить найближчого ворога.
2. Таймер автоатаки (`Timer` + `timeout`).
3. `projectile.tscn` (Area2D) з `direction: Vector2`, рухається в `_physics_process`.
4. Сигнал `area_entered` → урон → `queue_free()`.
5. `tower.tscn` з тими ж принципами + споживає енергію з `ResourceStore`.

**Перевірка:** ворог заходить у радіус вежі → вежа стріляє → ворог помирає.

---

## Блок 11 — Таймер день/ніч + хвилі (M8) (1 день)

1. `DayNightCycle` autoload або системна нода.
2. Два стани: `DAY` (5 хв), `NIGHT` (2 хв) + сигнал `phase_changed`.
3. 2D-освітлення або простий кольоровий `ColorRect`-оверлей.
4. `WaveSpawner` слухає `phase_changed == NIGHT` → спавнить мутантів на краю.

**Перевірка:** через 5 хв темніє, зі сходу біжать мутанти, вежі їх відстрілюють.

---

## Блок 12 — UI і Theme (наскрізна тема) (1 день)

Мета: UI-важка гра → треба вміти робити чисті панелі.

1. `Control`-ноди: `PanelContainer`, `VBoxContainer`, `HBoxContainer`, `Button`, `Label`, `TextureRect`, `ProgressBar`.
2. Anchors, margins, size flags — коли сам Godot їх не вгадує.
3. `Theme` як ресурс — кольори, шрифти, бордюри для всієї гри в одному місці.
4. Реюзабельна сцена віджета (`resource_row.tscn` — іконка + значення).

**Перевірка:** HUD зверху з 5 ресурсами виглядає однаково в будь-якому вирішенні екрана.

---

## Блок 13 — Діалогова система (M10) (1 день)

1. `DialogueData` (Resource) — масив `DialogueLine` (портрет, спікер, текст, варіанти).
2. `dialogue_ui.tscn` — модалка з плавним `Tween`-друкуванням тексту.
3. Пауза гри (`get_tree().paused = true`, у модалки `process_mode = ALWAYS`).
4. Тригер діалогу з `EventBus` (`game_started`, `first_hero_arrived`).

**Перевірка:** на старті гри Сержант говорить 3 фрази, дозволяє обрати 1 з 2 відповідей, потім гра розблоковується.

## Блок 14 — Збереження (M12) (1 день)

1. `FileAccess` + `JSON.stringify`.
2. Що зберігаємо: відкриті плитки, будівлі (тип+позиція+hp+lv), герої (hp, перки), `ResourceStore`, час гри.
3. `SaveSystem` autoload з методами `save()`, `load()`.
4. Версія формату (`"version": 1`) — щоб не ламати старі сейви.

**Перевірка:** зберіг → закрив гру → відкрив → усе на місці.
